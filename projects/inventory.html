Project 02

Multi-Platform Inventory Management System

A full-stack CRUD web application with real-time stock dashboards

// Overview

Every business tracks things — products, stock, orders, people. The Inventory Management System is a full-stack web application built around the real-world scenario of a "Game Store" — a fictional retailer stocking physical game copies, controllers, and accessories. The platform provides a clean dashboard where staff can add, edit, search, and delete inventory items, with live-updating stock level charts that give an at-a-glance warehouse overview.

The system is built on a React frontend communicating with a Node.js/Express backend API, backed by a MongoDB database. This project was a deliberate exercise in learning the full journey of data — from a MongoDB document, through an Express REST endpoint, fetched by React's state management, and finally rendered as a chart using Chart.js.

// What I Built
  • › A React SPA with full CRUD UI: item creation form, edit-in-place rows, delete confirmation modal, and search/filter bar.
  • › A RESTful Express API with five endpoints: GET /items, POST /items, PUT /items/:id, DELETE /items/:id, GET /items/stats.
  • › MongoDB schema with Mongoose for item validation (name, SKU, category, quantity, reorder threshold).
  • › A dashboard view with Chart.js bar/doughnut charts showing stock distribution by category and low-stock alerts.
  • › Custom SVG icons for item categories, created in Krita and exported for the web.
// What I Learned

State management in React becomes genuinely complex the moment CRUD operations must stay in sync with a remote database. This project was where the patterns of optimistic UI updates versus re-fetching-from-server became a real design decision with real trade-offs, not just a theoretical concept.

MongoDB's schema-less flexibility turned out to be a double-edged sword — early iterations had inconsistent document shapes that caused subtle frontend bugs. Adding Mongoose schema validation was the decisive fix and taught a lasting lesson about enforcing structure at the data layer regardless of database flexibility.

// The Challenge

Building charts that update without full page refresh when inventory data changes — keeping the chart in sync with the table view in real-time.

// The Approach

Lifted the inventory state to the top-level React context, making both the table and the chart derive from the same source of truth. When any CRUD operation resolves, it triggers a single context re-render that updates both views simultaneously — no separate fetch calls for the chart.
